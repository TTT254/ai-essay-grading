## 🚀 功能使用流程

### 教师工作流
1. 注册/登录教师账号
2. 查看班级列表
3. 创建作业任务
4. 等待学生提交
5. 查看AI批改结果
6. 审核并修改评分
7. 发布批改报告

### 学生工作流
1. 注册/登录学生账号
2. 选择所属班级
3. 查看作业任务列表
4. 提交作文（输入文本或上传图片OCR）
5. 等待AI批改
6. 查看批改报告
7. 与AI对话学习改进
8. 查看错题本总结常见错误

## 📝 测试数据说明

执行 `docs/insert-mock-data.sql` 后会创建：
- 10个教师账号（teacher1@test.com ~ teacher10@test.com）
- 30个学生账号（student701@test.com ~ student740@test.com）
- 7年级1-3班，每班10人
- 每个班级3-5个作业任务
- 部分学生已提交作文
- 部分提交已有批改报告和错题记录

**注意**: 这些账号只是在数据库中创建了记录，密码需要通过 Supabase Auth 注册才能生成。建议：
1. 先执行 SQL 创建数据
2. 然后通过前端注册对应的邮箱账号

## 🔑 AI 功能配置

### 1. 配置环境变量
在 `backend/.env` 中添加：
```env
DASHSCOPE_API_KEY=your_api_key_here
```

### 2. 获取 API Key
访问：https://dashscope.aliyun.com/
注册账号并获取 API Key

### 3. 测试 AI 功能
- 访问学生端提交作文
- 查看是否能自动批改
- 测试 AI 对话功能

```
前端 (React + TypeScript + Vite)
├── 学生端
│   ├── 任务列表
│   ├── 提交作文
│   ├── 查看报告
│   ├── 错题本
│   └── AI对话
├── 教师端
│   ├── 班级管理
│   ├── 作业管理

│   └── 数据统计
└── 认证
    ├── 登录
    ├── 注册
    └── 邮箱确认

后端 (Python + FastAPI)
├── API路由
│   ├── /api/auth - 认证
│   ├── /api/student - 学生端
│   ├── /api/teacher - 教师端
│   ├── /api/grading - AI批改
│   ├── /api/ai-chat - AI对话
│   └── /api/mistakes - 错题本
├── AI服务
│   └── 阿里云通义千问
└── 数据库
    └── Supabase (PostgreSQL)
```
